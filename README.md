## Reproduction of the LIME-FOLD Algorithm (Heart Disease Dataset)

This project reproduce the AAAI-19 paper. 
**“Induction of Non-Monotonic Logic Programs to Explain Boosted Tree Models Using LIME”**  
The LIME-FOLD experimental procedure and results on the UCI Heart Disease (Cleveland) dataset (heart row in Table 1).
---

## program structure

- **`src/`**：Source code
  - **`data_processing/`**
    - `heart_preprocessing.py`：Step 1，Download UCI heart data、preprocessing、MDLP discretization、training XGBoost，then：
      - `data/processed/X_final.csv`
      - `data/processed/y_binary.csv`
      - `results/models/xgb_model.json`
      - `results/figures/xgb_feature_importance.png`（Step1 Feature importance figure）
    - `mdlp_discretizer.py`：Fayyad & Irani (1993) MDLP discretization achieved
  - **`explanation/`**
    - `lime_explanations.py`：Base on Step1 XGBoost model，applied LIME for each sample, generate an explanation, and save it：
      - `results/models/lime_explanations.pkl`
  - **`models/`**
    - `fold_algorithm.py`：FOLD algorithm implementation (corresponding to the formula in the paper Algorithm 3 FOIL IG)
    - `lime_fold_main.py`：LIME‑FOLD main process：
      - According to Algorithm 4 on Paper，transform LIME interpretation + preprocessed data into ILP background knowledge `BK`, positive and negative examples `E+ / E-`
      - Running FOLD via transformed dataset，use **5‑fold cross-validation** for evaluating Precision / Recall / Accuracy / F1
- **`experiments/`**
  - `run_heart_experiment.py`：Complete script for reproducing the experiment (Steps 1–5, one-click run)
- **`data/`**
  - `raw/`：Raw data (not under version control)
  - `processed/`：Preprocessing results (automatically generated by Step 1)
- **`results/`**
  - `models/`：Intermediate products such as XGBoost Model and LIME interpretation
  - `figures/`：Experimental figures (Step 1 Feature importance; Table 1 Comparison figures)
  - `logs/`：Experimental result（CSV）

## Operating steps (complete experiment reproduction)

All commands are executed in the project root directory by default, and the virtual environment is already activated.

1. **Step 1：Preprocessing + XGBoost Training + Saving the Model**

   ```bash
   python -m src.data_processing.heart_preprocessing
   ```

   The script will:

   - Download the Heart Disease (Cleveland) dataset from the UCI repository；
   - Impute missing values ​​using the median；
   - Discretize the numerical features (`blood_pressure`, `serum_cholestoral`, `oldpeak`, `major_vessels`) using MDLP；
   - Perform one-hot encoding on the categorical features and rename them to the semantic names used in the paper (e.g., `chest_pain_4`, `thal_7`, etc.)；
   - Train an XGBoost binary classification model on the complete dataset；
   - Save the preprocessed results and the model; Plot and save the feature importance plot：
     - `results/figures/xgb_feature_importance.png`。

2. **Step 2–5：Generating LIME interpretations + LIME-FOLD + comparison with the paper**

   ```bash
   python experiments/run_heart_experiment.py
   ```

   This script will automatically complete the process according to the paper's workflow:

   - **Step 2**：If not exist `results/models/lime_explanations.pkl`，then call
     `generate_lime_explanations` XGBoost model based on Step 1 generates a LIME explanation for each sample.；
   - **Step 3**：call `run_lime_fold_experiment`：
     - According to the paper Algorithm 4, LIME interpretation is converted into ILP background knowledge `BK` and positive/negative`E+, E-`；
     - Run FOLD（Algorithm 3）on transformed dataset，learn default rules and exception rules；
     - Use **5‑fold Stratified Cross‑Validation** calculate average Precision / Recall / Accuracy / F1；
   - **Step 4**：Save the numerical results of this experiment as
     `results/logs/lime_fold_results.csv`；
   - **Step 5**：Generate a bar chart comparing the bar chart with Table 1 of the paper：
     - `results/figures/experiment_comparison.png`  
       (The left image shows a comparison of F1 scores, and the right image shows a comparison of the number of rules.)。

After the script finishes executing, the terminal will output:

- The average Precision / Recall / Accuracy / F1 of this 5‑fold；
- Average number of rules (compared to approximately 6 rules in the heart dataset in the paper)；
- Paths to saved image and log files。

## Current reproducibility description

- The implementation is aligned with the key pseudocode in the paper:
  - FOIL Information Gain and FOLD Algorithm (Algorithm 3)；
  - Use LIME to generate local interpretations and perform dataset transformation according to Algorithm 4 (including positive/negative weighted features, numerical features are discretized using MDLP).
- The evaluation method uses the same **5-fold cross-validation** method as the paper, and the average index is output.
- The experimental results on the Heart Disease dataset (F1 score and number of rules) are quite close to those in the FOLD LIME row of Table 1 in the paper, but may not be completely consistent (possibly due to:
  - Specific hyperparameters in XGBoost,
  - MDLP Implementation and Parameters,
  - Randomness and Division Method  
and so on factors).

This repository primarily focuses on: **reproducing algorithm flow and interpretable rule forms**, and striving to approximate the metrics reported in academic papers, to facilitate teaching and further experiments.
